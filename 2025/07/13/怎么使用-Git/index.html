怎么使用 Git

什么是 Git

如果你有一个团队，每个人都要写代码，有随时可能会出现冲突，怎么办？

如果你随时需要回滚到上一个版本，怎么办？

这里就需要 Git 来管理你的代码了。

Git 是一个版本控制软件，它可以让你管理你的代码，包括：

  • 备份你的代码
  • 回滚到上一个版本
  • 比较不同版本的代码
  • 合并不同版本的代码
  • 创建分支
  • 推送代码到远程仓库
  • 拉取代码到本地

下载 Git

如果你使用 Windows 或 macOS，出门左转官网。
如果你使用 Linux，你可以使用 sudo apt install git 来安装 Git。

配置 Git

  1. 打开终端，执行 git config --global user.name "你的名字" 来设置你的名字。
  2. 执行 git config --global user.email "你的邮箱" 来设置你的邮箱。
  3. 打开你的远程仓库，如 GitHub，下面就以 GitHub 为例。
  4. 执行 ssh-keygen -t rsa -C "你的邮箱" 来生成 SSH 密钥。
  5. 打开 你的用户目录/.ssh/id_rsa.pub 文件，复制里面的内容。其中 你的用户目录 是你的用户目录，如 /home/你的用户名，或者 C:\Users\你的用户名。
  6. 复制里面的内容。
  7. 打开 GitHub，登录你的账号。
  8. 点击右上角的头像，选择“Settings”。
  9. 点击“SSH and GPG keys”。
  10. 点击“New SSH key”。
  11. 在“Title”中填写一个名字，如“我的电脑”。
  12. 在“Key”中粘贴你复制的内容。
  13. 点击“Add SSH key”。
  14. 选择一种验证方式，如“Password”。
  15. 接下来就可以使用 Git 来管理你的代码了。

使用 Git

  1. 打开终端，执行 git clone (你的仓库地址) 来克隆你的仓库。
  2. 进行修改。
  3. 执行 git add . 来添加修改的文件。
  4. 执行 git commit -m "你的提交信息" 来提交修改。
  5. 执行 git push origin master 来推送修改到远程仓库。（有可能是 git push origin main，要看你主分支名字）
  6. 如果你想看别人的修改，你可以执行 git pull 来拉取修改。
  7. 当然了，这只是 Git 的基础用法，还有很多高级用法，如：
    • 分支
    • 标签
    • 合并
    • 回滚
    • 查看日志
    • 查看状态
    • 查看差异
    • 查看远程仓库
    • 查看本地仓库
    • 查看提交
    • 查看分支
    • 查看标签

因为时间关系（笔者也不会），这里就不一一介绍了，感兴趣的可以自行探索。

完结撒花！
